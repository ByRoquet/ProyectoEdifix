Guía para Reportes de Mantenimiento

1

Incluir contexto claro

Describe el problema con suficiente detalle, incluyendo lo que esperabas que sucediera y lo que realmente ocurrió.

// Mal: "No funciona"
// Bien: "Al hacer clic en el botón de enviar, el formulario no se envía y no aparece ningún mensaje de error"
2

Proporcionar pasos para reproducir

Enumera los pasos específicos para que otro desarrollador pueda reproducir el problema.

1. Ir a la página de contacto
2. Rellenar todos los campos excepto el email
3. Hacer clic en "Enviar"
4. Observar: No hay validación ni mensaje de error
3

Incluir código relevante

Proporciona el fragmento de código donde ocurre el problema, usando bloques de código formateados.

// Función con el problema:
function validateForm() {
  const email = document.getElementById('email').value;
  // Falta validación de campo vacío
  if(!isValidEmail(email)) {
    showError('Email inválido');
  }
}
4

Mencionar el entorno

Especifica navegador, versión de Node.js, sistema operativo y cualquier dependencia relevante.

Navegador: Chrome 98.0.4758.102 (64-bit)
OS: Windows 10 Pro 21H2
Dependencias: React 17.0.2, Formik 2.2.9
5

Incluir mensajes de error

Si hay errores en la consola, inclúyelos completos, no solo "hubo un error".

Uncaught TypeError: Cannot read properties of null (reading 'value')
  at validateForm (form.js:25)
  at HTMLButtonElement.onclick (index.html:42)
6

Adjuntar capturas cuando sea útil

Una imagen puede mostrar el problema visual o el flujo donde ocurre.

[Captura del formulario con campo vacío]
[Captura de la consola con el error]
7

Documentar intentos de solución

Menciona lo que ya intentaste para resolver el problema.

- Verifiqué que el elemento con id 'email' existe en el DOM
- Intenté cambiar el selector por querySelector
- Probé en otro navegador con los mismos resultados
